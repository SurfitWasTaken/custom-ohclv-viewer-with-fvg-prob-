# Deployment Runbook — FVG Probability API

## Prerequisites

- Docker &amp; Docker Compose installed
- Trained model artifacts in `models/` (generated by Phase 3)
- Optimal strategy config in `models/phase5/optimal_config.json` (generated by Phase 5 optimizer)

---

## 1. Local Development

```bash
# Install dependencies
pip install -r requirements.txt

# Start the API
uvicorn src.production_api:app --host 0.0.0.0 --port 5000 --reload

# Start the monitoring dashboard (separate terminal)
python3 src/monitor_dashboard.py --port 5001
```

**Verify:**
```bash
curl http://localhost:5000/health
```

---

## 2. Docker Deployment

### Build &amp; Run

```bash
# Build the image
docker compose build

# Start the service (detached)
docker compose up -d

# Check status
docker compose ps
docker compose logs -f fvg-api
```

### Stop / Restart

```bash
docker compose down           # stop
docker compose restart fvg-api  # restart without rebuild
docker compose up -d --build    # rebuild and restart
```

---

## 3. API Usage

### Health Check

```bash
curl http://localhost:5000/health
```

Response:
```json
{
  "status": "healthy",
  "model_loaded": true,
  "n_features": 30,
  "optimal_config": { "target_mode": "partial", "stop_mode": "opposite" },
  "monitor": { "total_predictions": 0, "resolved": 0, "pending": 0 }
}
```

### Make a Prediction

```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "symbol": "EURUSD",
    "candles_1h": [ ... ],
    "candles_4h": [ ... ],
    "candles_daily": [ ... ]
  }'
```

Provide ≥200 1H candles, ≥50 4H candles, ≥30 daily candles. Each candle:
```json
{ "timestamp": "2024-01-01T00:00:00+00:00", "open": 1.10, "high": 1.101, "low": 1.099, "close": 1.1005, "volume": 100 }
```

### Record Outcome

```bash
curl -X POST "http://localhost:5000/outcome?prediction_id=pred_123&outcome=upper"
```

## 4. Monitoring

### Dashboard

```bash
python3 src/monitor_dashboard.py --port 5001
# Open http://localhost:5001
```

Auto-refreshes every 30s. Displays:
- Total / resolved / pending prediction counts
- Overall and rolling accuracy (last 50)
- Bias distribution (LONG / SHORT / NEUTRAL)
- Recent predictions table with outcomes

### Prediction Log

All predictions are appended to `models/phase5/prediction_log.jsonl`. Each line is a JSON record:
```json
{
  "prediction_id": "pred_1708900000000",
  "timestamp": "2024-02-26T00:00:00",
  "prob_upper": 0.73,
  "bias": "LONG",
  "confidence": 0.73,
  "outcome": "upper",
  "correct": true
}
```

### Drift Alerting

The `ModelMonitor` automatically checks rolling accuracy after each outcome is recorded. If accuracy drops below 55% over the last 50 resolved predictions, a drift alert is printed to stdout/logs.

## 5. Zero-Downtime Updates

### Model Update (no code change)

```bash
# 1. Replace model files in models/
cp new_xgboost_model.pkl models/xgboost_model.pkl

# 2. Restart the container (picks up new model on startup)
docker compose restart fvg-api
```

### Code Update

```bash
# 1. Pull latest code
git pull origin main

# 2. Rebuild and restart
docker compose up -d --build
```

Docker Compose `restart: unless-stopped` ensures the service auto-recovers from crashes.

## 6. Troubleshooting

| Issue | Solution |
|-------|----------|
| `No 1H data for EURUSD` | Ensure `data/raw/` contains CSV files |
| `ModuleNotFoundError` | Run `pip install -r requirements.txt` |
| API returns 500 | Check `docker compose logs fvg-api` |
| Dashboard shows 0 predictions | No predictions logged yet; POST to `/predict` first |
| Drift alert firing | Review recent outcomes; consider retraining model |
